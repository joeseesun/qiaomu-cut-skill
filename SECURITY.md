# Security Policy

Please report vulnerabilities privately through GitHub Security Advisories for this repository. Do not open a public issue for credential exposure, path traversal, command injection, unsafe media parsing, or provider upload/authorization bypasses.

Never include API keys, cookies, signed URLs, private captures, or copyrighted source media in a report. Use redacted logs and minimal synthetic fixtures.

The latest release on the default branch is supported. Provider behavior, third-party CLIs, ffmpeg builds, and local adapters remain separate trust boundaries.
