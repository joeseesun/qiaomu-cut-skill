# Contributing

Contributions are welcome through focused issues and pull requests.

1. Fork the repository and create a feature branch.
2. Keep changes minimal and preserve the governed sourcing, explicit confirmation, no-clobber, credential-redaction, and provenance rules.
3. Run `npm run validate` and `npm run release-check`.
4. Add or update smoke tests for behavior changes.
5. Explain user-visible behavior, verification evidence, licensing impact, and remaining limits in the pull request.

Do not commit downloaded movie clips, generated provider captures, credentials, system fonts, caches, renders, or user projects.
